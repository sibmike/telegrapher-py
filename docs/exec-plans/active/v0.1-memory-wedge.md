| 4.5 | ~~`ImportError` when LangChain extra missing~~ | dropped | ❌ removed |
| 4.6 | Tests | `tests/test_memory.py` (16) | ✅ |

### Acceptance

- ✅ `ConversationCompactor(level="L1", max_tokens=80)` walks the L1 → L3 → L5 ladder when a single round of compression doesn't fit budget.
- ✅ When even L5 doesn't fit, the buffer drops oldest turns rather than looping.
- ✅ Round-trip via `expand_on_load=True` returns NL `Message`s while leaving the underlying buffer in TE form.
- ✅ `compression_ratio()` returns 1.0 with no eviction; > 1.0 after eviction; 1.0 on empty buffer.
- ✅ `pytest tests/test_memory.py -v` passes (16/16).

### Decision: LangChain adapter dropped from v0.1

`langchain_core 1.x` removed `BaseMemory` and migrated memory to LangGraph's checkpointer pattern. Shipping a v0.1 adapter against the deprecated 0.x interface would be dead-on-arrival the moment a user upgrades, and pinning `langchain-core<1.0` would lock our release cadence to LangChain's deprecation cycle.

Decision: ship `ConversationCompactor` as a generic primitive in v0.1. Framework adapters (LangChain checkpointer, LlamaIndex, Haystack, LiteLLM) become opt-in v0.2 integrations once we have signal on which one matters most to real users. PS-001 and the README are updated to drop the "drop-in for ConversationSummaryMemory" framing.

## v0.1 done

All five phases complete: 97 passing tests, ruff + mypy clean across 21 source files. The package ships:

- `from telegrapher import Telegrapher` — `compress` / `expand` / `ratio` / `compress_stream`, mutex `model=` vs `model_in=`/`model_out=`, content-hash disk cache.
- `from telegrapher.memory import ConversationCompactor` — token-budgeted memory with the L1→L3→L5 eviction ladder.
- `from telegrapher.eval import validate` — corpus evaluator that returns an `EvalReport` and writes Markdown.
- `tg compress / expand / eval / download-model` CLI.

The package is feature-complete for v0.1 against `MockRunner`. Phase 6 (real `VLLMRunner` and `LlamaCppRunner` against fine-tuned weights) is the remaining item gating PyPI release.

## Risks & mitigations

| Risk | Mitigation |
|------|-----------|
| Fine-tuned weights not ready when v0.1 wants to ship | MockRunner gates the entire Phase 1–5 dev loop; release waits on weights, not on code |
| `vllm` Windows compatibility issues | `LlamaCppRunner` covers Windows/macOS/CPU; `vllm` is Linux+CUDA only by extras gating |
| LangChain API drift (we target `langchain-core>=0.2`) | Minimum-version pin + tests against the pinned floor |
| Real weights too large for users to download casually | 4-bit quantized GGUF (~5 GB) is the default download path |
